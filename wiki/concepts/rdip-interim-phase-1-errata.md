## Definition

**Rdip Interim Phase 1 Errata** refers to a formal correction or amendment package associated with the **January 2019 RDIP** documentation for **SR 132 West Expressway Phase 1**. In practice, an errata document records changes, clarifications, corrections, or replacements to information published in the original report or project package.

This type of errata is used to ensure that the project record reflects the most accurate and current information after the initial January 2019 issuance of the RDIP material.

## How it works

An errata document functions as a controlled update to an existing technical or planning document:

1. **Identifies the source document**  
   The errata references the original RDIP Phase 1 material, in this case the January 2019 package for SR 132 West Expressway Phase 1.

2. **Lists corrections or revisions**  
   It may modify text, figures, tables, plan sheets, notes, or technical statements that were found to be incomplete, outdated, or incorrect.

3. **Preserves document traceability**  
   Rather than replacing the original source entirely, errata provides a documented change history so readers can see what was altered and why.

4. **Supports downstream use**  
   Engineers, planners, reviewers, and decision-makers can rely on the corrected version when evaluating the project, while still maintaining the original issuance for recordkeeping.

In a project documentation context, errata is especially important when the underlying work involves regulatory review, design development, procurement, or public communication, because even small discrepancies can affect interpretation.

## Variants

Common variants of errata in project and technical documentation include:

- **Single-page correction sheets** — a concise list of changes to the original document.
- **Revision memoranda** — narrative explanations of the corrections and their basis.
- **Reissued document sets** — updated drawings or reports where the errata is incorporated directly into a new version.
- **Addenda** — supplemental updates issued before final adoption or approval.
- **Bulletins or clarifications** — narrow corrections focused on specific issues.

Depending on the project workflow, an errata may be distributed as a standalone file or embedded into a broader revision package.

## Trade-offs

Key considerations when using an errata document include:

- **Accuracy vs. continuity**  
  Errata improves correctness without discarding the original record, but readers must check both documents to understand the full picture.

- **Clarity vs. complexity**  
  Maintaining separate correction documents preserves traceability, but it can make document management more complex.

- **Version control burden**  
  Users must know whether they are reading the original, the corrected version, or both.

- **Risk of missed updates**  
  If an errata is not widely distributed or clearly linked, outdated information may continue to be used.

- **Auditability**  
  The main advantage is that changes remain transparent and reviewable, which is valuable in formal project documentation.
